# Hackathon Demo Runbook

> 用于现场演示和最终验收。目标是在 5 分钟内跑通“简历/JD -> 画像 -> 题目 -> 语音/文本答题 -> 报告 -> 复制”主闭环。

## 1. 演示前检查

| Check | Expected |
| --- | --- |
| `npm run typecheck` | Pass |
| `npm run dev` | 本地服务启动，默认 `http://localhost:3000`，端口占用时使用提示端口 |
| `.env.local` | 可选配置 Azure/LLM key，不在屏幕或日志中展示真实密钥 |
| 浏览器权限 | 麦克风可选；不支持 STT 时使用手动编辑 |
| 网络 | 不稳定时仍可使用演示兜底样例包 |

## 2. 标准演示路径

1. 打开正式 Next.js 应用。
2. 点击“填充并生成画像”，或先点击“一键填充：AI 产品经理实习生”后手动修改输入再生成画像。
3. 如需现场稳定演示，开发环境可在 `Demo Ops` 面板选择“强制演示兜底”。
4. 展示画像中的匹配点、风险点、关键词。
5. 点击“生成 3 道题”。
6. 展示 3 道题的题目、意图和 expected signals。
7. 点击“开始答题”。
8. 选择一题播放 TTS：Azure 可用则播放 Azure，不可用则展示 Web Speech 兜底或文本继续。
9. 点击“填入样例答案”。
10. 点击“生成复盘报告”。
11. 展示最终报告、Top 风险、行动项、每题报告卡片。
12. 点击“一键复制”，确认复制成功提示。
13. 展示可复制内容包含“优化答案”和“复盘报告”。

## 3. 必演兜底路径

| Scenario | 操作 | Expected |
| --- | --- | --- |
| 空输入 | 不填简历/JD，点击生成画像 | 停留 setup，提示简历和 JD 不能为空 |
| TTS 不可用 | Azure 未配置或播放失败 | 文本仍可继续，Web Speech 或 unsupported 状态不阻断流程 |
| STT 失败 | 点击“模拟 STT 失败” | 保留答案文本，可重试或手动编辑 |
| 缺失答案 | 不填答案直接生成报告 | 报告标记缺失，不生成假回答 |
| 报告生成失败 | 注入不可达 LLM provider | 保留问题和答案，显示重试和演示兜底 |
| 剪贴板失败 | 点击“验证剪贴板失败兜底” | 展示手动复制文本框并选中内容 |

## 4. 开发故障注入

开发环境页面顶部会显示 `Demo Ops` 面板。生产构建不展示该面板。

| Control | Expected |
| --- | --- |
| 强制演示兜底 | profile/questions/report 直接返回演示数据，避免现场网络影响 |
| LLM 失败 | API 返回 retryable failure，前端保留输入并展示演示兜底入口 |
| TTS 失败 | Azure TTS 不可用，文本流程继续，可切 Web Speech |
| STT 失败 | 识别进入失败态，已输入文本保留，可重试或手动编辑 |
| Clipboard 失败 | 一键复制进入手动复制兜底 |

## 5. 讲解要点

| Section | Talk Track |
| --- | --- |
| 契约稳定 | 所有接口统一 `CommonResponse`，问题、答案、报告通过 `questionId` 关联 |
| Demo 兜底 | 无 LLM key、网络失败、语音失败时主流程仍能完成 |
| 语音层 | Azure TTS 优先，Web Speech 兜底；STT 失败可重试和手动编辑 |
| 报告层 | 每题包含评分、风险标签、致命问题、优化答案、60 秒口述版 |
| 复制层 | 一键复制覆盖优化答案和复盘报告，剪贴板权限失败时可手动复制 |

## 6. 发布前最终验收建议

Figma 接入后执行：

1. 三种面试官风格各跑一次完整流程。
2. 使用真实 LLM key 检查题目是否体现简历/JD/风格。
3. 桌面和手机 H5 各录制一遍主流程。
4. 故障注入验证：LLM provider failed、TTS failed、STT failed、clipboard failed。
5. 检查页面和日志不输出 `.env.local` 中真实密钥。
