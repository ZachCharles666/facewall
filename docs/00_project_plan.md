# 项目规划：面试嘴替教练 Hackathon MVP

## 1. 业务目标

帮助求职者把简历/JD 转成有针对性的面试训练：系统模拟不同风格的面试官提问，用户作答后获得结构化复盘、优化答案和可复制话术。

## 2. 核心用户

| 用户 | 场景 | 主要诉求 |
| --- | --- | --- |
| 应届生 | 校招前准备 | 不知道面试官会怎么问，需要快速练习 |
| 转岗求职者 | 解释过往经历和目标岗位关系 | 需要把经历包装成岗位相关表达 |
| 社招候选人 | 针对 JD 做专项准备 | 需要识别风险点和补强回答结构 |

## 3. 主闭环

1. 用户输入或粘贴简历/JD，也可以使用演示样例包。
2. 用户选择面试官风格。
3. 系统解析候选人画像。
4. 系统生成 3 道面试题。
5. 系统用 TTS 朗读题目，同时展示题目文本。
6. 用户语音或文本作答。
7. 系统生成复盘报告、优化答案、60 秒口述版。
8. 用户一键复制优化答案和报告。

## 4. 目标阶段

| 项 | 定义 |
| --- | --- |
| 目标阶段 | Alpha-Demo |
| Alpha 部分 | 主流程、接口契约、异常兜底、可验证报告结构 |
| Demo 部分 | 面试官头像、声波动效、语音风格、演示样例 |
| 不追求 | 登录、历史记录、真实复杂音频处理、实时多轮追问 |

## 5. 本轮范围

| 模块 | 本轮必须做 | 可降级方式 | 延后 |
| --- | --- | --- | --- |
| 简历/JD 输入 | 是 | 演示样例包一键填充 | 文件解析、简历上传 |
| 面试官风格 | 是 | 固定 3 个配置 | 更多风格市场 |
| 候选人画像 | 是 | LLM 失败时用演示兜底 | 长期画像存储 |
| 题目生成 | 是 | 演示题目兜底 | 自由多轮追问 |
| TTS 提问 | 是 | Azure 失败降级 Web Speech 或纯文本 | 真实口型/波形 |
| 语音答题 | 是 | STT 失败允许重试和编辑 | 音频文件上传 |
| 报告生成 | 是 | 非流式 JSON | 流式逐段生成 |
| 一键复制 | 是 | 复制纯文本 | 富文本/多平台格式 |
| UI 设计替换 | 是 | 先以稳定组件骨架接入 | 完整设计系统 |

## 6. 关键产品确认结果

| 问题 | 当前结论 |
| --- | --- |
| 面试官风格数量 | 先做 3 个 |
| 报告是否必须流式 | 不必须，按开发成本选择 |
| 复制什么内容 | 复制优化答案和报告 |
| 语音答题失败怎么办 | 优先重试，保留已识别文本，允许手动编辑 |
| 演示样例谁提供 | 产品允许开发侧定义 |

## 7. 待产品/设计继续确认

| 项 | 为什么需要确认 | 建议默认值 |
| --- | --- | --- |
| 三个面试官最终命名和文案 | 影响 UI 和 prompt 风格 | 沿用当前 3 个 |
| 分数是否展示总分和分项分 | 影响报告布局 | 总分 + 6 个维度 |
| 风险标签数量 | 影响卡片密度 | 每题 2-3 个 |
| 60 秒口述版长度 | 影响生成约束 | 中文 180-240 字 |
| Figma 交付范围 | 影响前端拆分顺序 | 首屏、面试页、报告页、状态组件 |

## 8. 执行阶段

| 阶段 | 目标 | 产出 |
| --- | --- | --- |
| Phase 0 | 契约和规划冻结 | docs、数据结构、todo |
| Phase 1 | Next.js 本地闭环 | 页面骨架、演示样例、状态机 |
| Phase 2 | Next.js API 和 AI 接口 | Route Handlers、prompt、JSON 校验、兜底 |
| Phase 3 | 语音能力接入 | Azure TTS、Web Speech fallback、STT |
| Phase 4 | 报告和复制体验 | 报告卡片、复制、异常处理 |
| Phase 5 | 设计稿替换和验收 | Figma 视觉落地、全链路验收 |

## 9. 技术栈决定

| 项 | 决定 |
| --- | --- |
| 正式前端 | Next.js App Router + React + TypeScript |
| API 层 | Next.js Route Handlers |
| 样式 | 先用 CSS Modules 或 Tailwind 二选一；Figma 到位后 token 化 |
| 语音代理 | Azure TTS 走服务端 Route Handler，禁止前端暴露 key |
| 现有 demo | `tts-demo` 保留为语音效果验证样板，不作为正式产品结构 |
| 数据持久化 | 本轮不接数据库，session/local state + 演示样例包 |
